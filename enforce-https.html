"Enforce HTTPS" is Grayed Out?

Here's why the toggle is disabled and how to turn it back on.

Why the toggle is unavailable

GitHub Pages only enables Enforce HTTPS after it can issue an SSL/TLS certificate for your custom domain. If your DNS records are not pointing to GitHub Pages, the certificate request fails or is still pending, and the checkbox stays disabled.

Common causes include: the domain still points to another host, mixed A/AAAA records that don't match GitHub's IPs, or a proxy (like an orange-clouded Cloudflare record) intercepting traffic.

How to fix it

1) Point DNS to GitHub Pages. For apex domains, use A/AAAA records for GitHub Pages IPs. For subdomains, use a CNAME to USERNAME.github.io.

2) Wait for the certificate. After DNS resolves to GitHub Pages, GitHub automatically provisions the certificate. This can take a few minutes after propagation.

3) Avoid conflicting proxies. Pause SSL/HTTP proxies or CDN features that rewrite traffic until the GitHub certificate is issued, then re-enable them if needed.

What to expect

Once DNS is correct and the certificate is issued, the "Enforce HTTPS" checkbox becomes clickable. If it stays disabled after DNS is fixed, remove the custom domain in Pages settings, add it back, and let GitHub retry the certificate request.
